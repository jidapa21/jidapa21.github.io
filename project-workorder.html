Work Order Management Application

An end-to-end web application that digitalizes and streamlines manufacturing work orders — from creation and assignment through execution, tracking, and reporting.

Overview

Project Overview

This project is a full-stack work order management system developed for The Nawaloha Industry Co., Ltd. to replace paper-based and disconnected workflows. The application digitalizes the complete lifecycle of a manufacturing work order — from supervisor creation and technician assignment to task execution, sign-off, and reporting.

Built with C# and SQL Server Express, the system provides a structured, auditable trail of all maintenance and production tasks while giving managers real-time visibility into open workloads.

Problem Statement

The Problem

Work orders at the facility were managed through paper forms and verbal communication, creating several pain points: lost records, unclear ownership of tasks, no visibility into WO status for supervisors, and difficulty generating completion reports for compliance.

Technicians often received conflicting priorities, and there was no centralized system to track time-to-completion or identify recurring equipment issues across work order history.

Solution Approach

How I Solved It

I designed and developed the application using an Agile/Scrum methodology, iterating through sprints with regular feedback from facility supervisors and technicians:

  • Designed a normalized relational database schema in SQL Server Express to model WO lifecycle stages, users, equipment, and task history
  • Built a RESTful API backend in C# exposing endpoints for CRUD operations on work orders, assignments, and status updates
  • Implemented role-based access control for Supervisors, Technicians, and Administrators
  • Created structured forms for WO creation with equipment lookup, priority levels, and due date assignment
  • Built a dashboard showing open WOs by priority, technician workload distribution, and overdue items
  • Generated PDF/export summaries for management reporting and compliance records
Key Features

Key Features

  • Full work order lifecycle: Create → Assign → In Progress → Review → Close
  • Role-based access control with Supervisor, Technician, and Admin roles
  • Equipment registry with maintenance history linkage
  • Priority classification (Critical, High, Medium, Low) with visual indicators
  • Real-time status dashboard with overdue alert highlighting
  • Digital sign-off and completion notes for each WO stage
  • Reporting exports for compliance and performance review
Result / Impact

Results & Impact

The application eliminated paper-based work order handling at the facility, creating a fully auditable digital record of all maintenance and production tasks. Supervisors gained real-time visibility into task status and technician workloads for the first time. The system's history database enabled analysis of recurring equipment issues, supporting preventive maintenance planning.
